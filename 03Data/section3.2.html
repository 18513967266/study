1. 元素等待

1.1 什么是元素等待？

概念：WebDriver定位页面元素时如果未找到，会在指定时间内一直等待的过程；

1.2 为什么要设置元素等待？

1. 由于网络速度原因
2. 电脑配置原因
3. 服务器处理请求原因

WebDriver元素等待有几种类型呢？

1.3 元素等待类型

1. 显式等待
2. 隐式等待

2. 显式等待

概念：使WebDriver等待指定元素，如果能定位到元素则直接返回该元素，不再等待；
      如果不能定位到该元素，则间隔一段时间后再去定位元素；
      如果在达到最大时长时还没有找到指定元素，则抛出超时异常(TimeoutException)

提示：
    1). 在WebDriver中把显式等待的相关方法封装在WebDriverWait类中
    2). 判断元素是否存在的方法封装在expected_conditions模块中

2.1 案例-1 注册页面A

需求：
    1. 如果用户名文本框存在，就输入admin

2.2 实现难点分析

1. 导包 等待类         --> from selenium.webdriver.support.wait import WebDriverWait
2. 导包 判断条件     --> from selenium.webdriver.support import expected_conditions
3. WebDriverWait(driver, timeout, poll_frequency=0.5)
        1). driver：浏览器对象
        2). timeout：超时的时长，单位：秒
        3). poll_frequency：检测间隔时间，默认为0.5秒
4. 调用方法 until(method)：直到..时
        1). method：调用expected_conditions.presence_of_element_located(locator)
                    locator：元素定位器，该参数是一个元组，包括定位类型和定位内容。如：(By.id, 'user')
5. element = WebDriverWait(driver, 10, 1)\
        .until(expected_conditions.presence_of_element_located((By.ID, "user")))

2.3 案例-1 代码示例

import time

from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support import expected_conditions
from selenium.webdriver.support.wait import WebDriverWait

driver = webdriver.Firefox()
driver.get(r"D:\webAutoTest\page\注册A.html")

element = WebDriverWait(driver, 10, 1)\
    .until(expected_conditions.presence_of_element_located((By.ID, "userA")))
element.send_keys("admin")

time.sleep(3)
driver.quit()

3. 隐式等待

说明：等待元素加载指定的时长。如果元素定位失败，那么就会触发隐式等待有效时长，如果在指定时长内定位到该元素
就直接返回该元素，否则抛出NoSuchElementException异常。如果元素在第一次就定位到则不会触发隐式等待时长；

显式与隐式区别：
    1. 作用域：显式等待为单个元素有效，隐式为全局元素
    2. 方法：显式等待方法封装在WebDriverWait类中，而隐式等待则直接通过浏览器实例化对象调用

3.1 隐式等待调用方法

方法：driver.implicitly_wait(timeout)
      (timeout：为等待最大时长，单位：秒)
